# ЛР 7. FastAPI, разработка API приложение

## Цель лабораторной работы

Реализовать API для функционала приложения по варианту

## Введение

- FastAPI - популярный асинхронный фреймворк для API
- Pydantic - валидация/сериализация данных.
- Uvicorn - библиотека для веб-сервера
- SQLAlchemy - библиотека для работы с реляционными СУБД
- asyncpg - библиотека асинхронных запросов в Postgresql

Информация про REST https://practicum.yandex.ru/blog/chto-takoe-rest-api-i-kak-rabotaet/

## Требования лабораторной работы

- Ваше API использует асинхронное подключение к PostgreSQL через asyncpg
- Ваши API cтроит ORM-модели на SQLAlchemy 2.0
- Ваш API применяет валидацию и сериализацию через Pydantic (v2)
- Ваш REST API доступен через FastAPI
- Ваш API запускается через uvicorn (например: uvicorn main:app --reload), либо поднимается в докер контейнере

## requirements.txt
```
fastapi
pydantic
SQLAlchemy
uvicorn
asyncpg
```

### Вариант 1

**Описание**

Асинхронное API для создания, обновления и отслеживания задач с возможностью назначать исполнителей и устанавливать сроки.

**Основные сущности:**

- user (id, name, email)
- task (id, title, description, status, due_date, assignee_id → User)

**Эндпоинты:**

- Создание задачи и пользователей
- Обновление задачи и пользователей
- фильтрация по статусу
- получение задач пользователя.

### Вариант 2

**Описание**: 

API для публикации статей, добавления комментариев и управления авторами.

**Основные сущности:**

- author (id, username, bio)
- post (id, title, content, created_at, author_id)
- comment (id, content, post_id, author_id, created_at)

**Эндпоинты:**

- создание постов/комментариев
- получение постов с вложенными комментариями
- поиск по автору.

### Вариант 3

**Описание**

Минималистичный API интернет-магазина для управления товарами и заказами.

**Основные сущности:**

- customer (id, name, email)
- order (id, customer_id, desc, created_at, status)

**Эндпоинты:**

- оформление заказа(создание)
- получение заказа
- удаление заказа
- история заказов клиента.

### Вариант 4

**Описание**

API для резервирования переговорок в офисе на определённое время.

**Основные сущности:**

- room (id, name, capacity, location)
- booking (id, room_id, user_name, start_time, end_time)

**Эндпоинты:**

- создание/удаление бронирования
- проверка доступности комнаты на интервал времени
- список бронирований пользователя.

### Вариант 5

**Описание:**

Пользователи регистрируют привычки (например, «пить воду», «делать зарядку») и отмечают их выполнение по дням.

**Основные сущности:**

- habit (id, user_id, name, description)
- completion (id, habit_id, date)

**Эндпоинты:**

- создание привычки
- отметка выполнения
- получение статистики (сколько дней подряд выполнено и т.п.).

### Вариант 6

**Описание:**

API для ведения личной библиотеки: добавление книг, отслеживание прогресса чтения.

**Основные сущности:**

- book (id, title, author, isbn, total_pages)
- readingEntry (id, book_id, user_id, current_page, updated_at)
- user (id, name)

**Эндпоинты:**

- добавление книги
- обновление текущей страницы
- получение списка непрочитанных/прочитанных книг.
